# 残差サイノグラムのHFエネルギー最小化によるテンプレートマッチング

## 1. 目的

ラドン変換のサイノグラムを用いた回転不変テンプレートマッチングにおいて、自然画像の背景がマッチングを阻害する問題を解決するため、残差サイノグラムの高周波エネルギー最小化という新しいアプローチを検討する。

## 2. 背景と問題

### 2.1 既存手法の限界

現在のハフ投票ベースの手法は、テンプレートが画像の主要コンテンツである場合（面積比≥25%）に良好に動作するが、自然画像背景での小面積テンプレート検出には失敗する。

根本原因: サイノグラムは画像全体の線積分であり、テンプレートの面積比が小さいと背景がサイノグラムを支配する。

### 2.2 検討した手法の推移

| # | アプローチ | 結果 | 失敗の原因 |
|---|-----------|------|-----------|
| 1 | 360度ハフ投票 | 180度曖昧性は解消 | 自然画像では投票が散逸 |
| 2 | 勾配画像前処理 | 一部改善 | 自然画像ではエッジが遍在 |
| 3 | 負の差分ペナルティ | 機能せず | サイノグラムのスケール差で常に正 |
| 4 | サイノグラム行の正規化 | 悪化 | NCC入力の追加正規化が有害 |
| 5 | HP+NCC（行方向ハイパスフィルタ） | 未完了 | 計算時間の問題で中断 |
| 6 | 列方向FFT基本成分 | 背景に埋没 | 線積分の性質で背景が支配 |
| 7 | α(θ)滑らかさ指標 | 識別力不足 | 背景がフィット係数を支配 |
| **8** | **残差HFエネルギー最小化** | **成功** | — |

## 3. 理論

### 3.1 サイノグラム減算の物理的意味

ターゲット画像にテンプレートが置換配置されている場合:
- テンプレート領域: `image(x,y) = template(x,y)`
- テンプレート外: `image(x,y) = background(x,y)`

ラドン変換の線形性より:
```
S_image = S_template_at_pose + S_background_with_hole
```

正しい姿勢 `(α, dx, dy)` でテンプレートのサイノグラムを減算すると:
```
S_residual = S_image - S_template_at_pose = S_background_with_hole
```

これを逆ラドン変換すると「テンプレート領域が黒い穴の背景画像」が復元される。

### 3.2 正しい減算 vs 間違った減算

| | 残差の画像空間表現 | 特徴 |
|---|---|---|
| 正しい減算 | 穴あき背景（滑らかな背景 + 矩形の穴） | 低周波が支配的 |
| 間違った減算 | テンプレートの残留 + 偽ゴースト + アーティファクト | 高周波成分が増大 |

間違った減算では:
1. 正しい位置のテンプレートが除去されない（残留）
2. 間違った位置から信号が引かれる（ゴースト）
3. これらの不整合がアーティファクトとして画像全体に影響

### 3.3 HFエネルギー最小化

Parsevalの定理により、画像空間のエネルギーはサイノグラム空間のエネルギーと等価:
```
Σ|f(x,y)|² ∝ Σ|S(θ,t)|²
```

残差サイノグラムの各行にハイパスフィルタを適用し、高周波エネルギーを計算:
```
E_HF(α, dx, dy) = Σ_θ Σ_t |HPF[S_image(θ,t) - S_template(θ,t; α,dx,dy)]|²
```

このE_HFを最小化する `(α, dx, dy)` が正しいテンプレート姿勢。

**重要**: 全体エネルギー最小化ではHFエネルギー最小化のようにうまくいかない。全体エネルギー `Σ|S_img - S_tmpl|²` を展開すると `Σ|S_img|² - 2·Σ(S_img·S_tmpl) + Σ|S_tmpl|²` となり、これは単なるサイノグラム相互相関の最大化に帰着し、背景との相関に支配される。HFエネルギーに限定することで、アーティファクト由来の高周波成分だけを評価対象にできる。

### 3.4 テンプレートが画像内に収まる制約

テンプレートが画像境界の外にはみ出す場合、サイノグラムへの寄与自体が小さくなり、HFエネルギーが自動的に低下する偽の最小値が生じる。回転後のバウンディングボックスが画像内に完全に収まることを制約条件として加える必要がある。

## 4. 実験結果

### 4.1 実験条件

- テンプレート: 犬の鼻部分のクロップ (64x64)
- ターゲット: 自然画像背景 (128x128) にテンプレートを回転配置
- 面積比: 約25% (64x64 / 128x128)
- 正解: angle=30°, dx=15, dy=-10
- HPFカットオフ: 行のFFTの低周波12.5%をゼロ化

### 4.2 結果

| パラメータ | 正解 | HF energy最小 | 誤差 |
|---|---|---|---|
| angle（1D sweep） | 30° | 30° | 0° |
| dx（1D sweep） | 15 | 15 | 0 |
| dy（1D sweep） | -10 | -10 | 0 |
| angle x dx（2D sweep） | (30°, 15) | (30°, 16) | (0°, 1px) |
| dx x dy（2D sweep） | (15, -10) | (16, -10) | (1px, 0) |

### 4.3 比較: 全体エネルギー最小化 vs HFエネルギー最小化

| 手法 | angle | dx | dy |
|---|---|---|---|
| 全体エネルギー最小化 | 53° (err=23°) | -32 (err=47) | 32 (err=42) |
| HFエネルギー最小化 | **30° (err=0°)** | **15 (err=0)** | **-10 (err=0)** |

## 5. 考察

### 5.1 なぜHFエネルギーが機能するか

1. **正しい減算**: 残差は「穴あき背景」であり、穴の境界を除けば滑らかな画像。高周波成分が少ない
2. **間違った減算**: テンプレートの残留とゴーストが画像全体にアーティファクトを生じさせ、高周波成分が増大する
3. **背景の影響が軽減**: 全体エネルギーでは背景のDC成分・低周波成分が支配するが、HFフィルタによりこれらが除去される

### 5.2 位置と角度の結合問題

HFエネルギーは位置と角度の**同時評価関数**であり、片方を固定してもう片方を探索する交互最適化では不安定になる。

原因はFFTの位相依存性にある:
- 画像の平行移動は `FFT(a')[k] = FFT(a)[k] · exp(-jφ)` と位相回転を引き起こす
- `|FFT(a)|` （振幅）は平行移動不変だが、`|FFT(a) - FFT(b)|`（複素差分の振幅）は位相に依存
- したがって `|z·exp(-jφ) - w| ≠ |z - w|` が一般に成立し、位置が違えばHFエネルギーも変わる

実験で確認した結果:
- 角度を正解に固定 → 位置は正しく検出できる（err=0〜1px）
- 位置を正解に固定 → 角度は正しく検出できる（err=0°）
- **位置を(0,0)に固定して角度を探索 → 角度が大きく外れる（180°誤差など）**
- 角度が外れた状態で位置を探索 → 位置も外れる → 回復不能

### 5.3 サイノグラムのシフト演算による高速化の検討

ラドン変換の性質として:
- **回転 α**: サイノグラムの行をα行シフト `S_rot[θ] = S_orig[(θ+α) % 360]`
- **平行移動 (dx,dy)**: 各行をt方向に `dx·cos(θ) - dy·sin(θ)` だけシフト

これによりラドン変換はテンプレート1回 + ターゲット1回の計2回で済むはずだが、以下の問題が判明:

1. **離散ラドン変換の角度依存性**: 既存の`perpendicularSum`は`int()`による座標丸めを使用しており、角度ごとに離散化パターンが異なる。そのため行シフトで回転を表現すると誤差が生じる
2. **バイリニア補間版を実装**（`radonTransformInterp`）: 座標をfloatのまま保持しバイリニア補間で画素値を取得。これにより行シフトの精度が改善し、**角度のみの検出（位置=正解固定）では全角度で誤差0°**を達成
3. **行シフトの符号**: `warpAffine(-angle)`による時計回り回転に対応するシフトは `(θ+α) % 360`。当初 `(θ-α)` としていたため全角度で `360-α` が検出される問題があった

しかし、5.2の結合問題により、位置を未知としたまま行シフトで角度を検出することはできなかった。

### 5.4 角度のみの検出の試み

位置推定を棚上げし、角度検出だけでも行シフトベースで実現できないか検討した。

**行ごとHFエネルギーの利用**: 画素(dx,dy)のサイノグラム軌跡 `t(θ) = dx·cos(θ) - dy·sin(θ)` はθ=0〜359で必ず2回t=0を横切る。正しい角度で行シフトしたとき、この2行では位置(0,0)で減算しても正しく相殺される。

試した指標:
| 指標 | 原理 | 結果 (7ケース中OK) |
|---|---|---|
| min(row HF) | 全行中の最小HFエネルギー | 2/7 |
| min(180°pair) | θとθ+180°のHFペア和の最小 | 2/7 |
| mean(bottom-3 pairs) | 上位3ペアの平均 | 2/7 |

360行中2行の信号ではSNRが不足し、背景ノイズに埋もれる。

### 5.5 計算コスト

位置と角度を同時に探索する場合の計算量:
- 全探索: `360 × dx_range × dy_range` — 通常の総当たりテンプレートマッチングと同等以上
- 交互最適化: 結合問題のため不安定
- 行シフトによる高速化: 角度検出の精度不足で実用困難

128x128画像でのベンチマーク:
| 手法 | ラドン変換回数 | 実行時間 |
|---|---|---|
| 全候補ラドン変換（最適化前） | ~2200回 | ~58s |
| 角度=ラドン, 位置=シフト | ~360回 | ~20s |
| 全シフト（ラドン2回のみ） | 2回 | ~8s（ただし精度不足） |

## 6. 結論

### 6.1 HFエネルギー最小化の位置づけ

残差サイノグラムのHFエネルギー最小化は、**正しい位置と角度を同時に与えれば**自然画像背景でも正確にテンプレートの姿勢を検出できる（全テストケースで誤差0）。しかし、位置と角度の分離探索は位相依存性により本質的に困難であり、同時探索は計算量が通常の総当たりと同等以上になる。

### 6.2 手法の適用範囲

| 条件 | ハフ投票 | HFエネルギー |
|---|---|---|
| テンプレート面積比 ≥ 25% | 高速・高精度 | 高精度だが低速 |
| 自然画像背景・小面積比 | 精度低下 | 原理的に可能だが計算コスト大 |

既存のハフ投票ベースの手法は、テンプレートが画像の主要コンテンツである場合に最適であり、この制約の下で実用的。

### 6.3 得られた知見

1. `|FFT(a) - FFT(b)| ≠ |FFT(a)| - |FFT(b)|` — 振幅の差分と差分の振幅は異なる（FFTの非線形性）
2. 平行移動はFFT位相を回転させ、複素差分の振幅を変化させる — HFエネルギーが位置に敏感な理由
3. 離散ラドン変換では`int()`丸めが角度依存の離散化誤差を生む — バイリニア補間版で改善可能
4. サイノグラムの行シフトによる回転表現は、補間版ラドン変換 + 正しい符号 `(θ+α)%360` で正確に成立
5. 画素の正弦波軌跡のt=0交差（180°間隔の2点）は理論的に正しいが、自然画像背景のSNR下では検出困難

## 7. 関連する実験の記録

### 7.1 列方向FFT基本成分の分析

サイノグラムの各列（固定オフセットtでの角度θ変化）をFFTし、基本成分（ω=1）の振幅A(t)と位相φ(t)を分析した。

- 均一背景: A(t)がテンプレート位置に局在、φ(t)が回転を反映
- 自然背景: 背景の基本成分がテンプレートを圧倒し、位相変化が埋没

結論: 列方向FFTはサイノグラム全体を使うため、背景支配の問題は行方向マッチングと同じ。

### 7.2 サイノグラム減算の復元画像

サイノグラムの素の減算（スケーリングなし）を行い逆ラドン変換で画像を復元:
- 正しい減算: テンプレート領域が黒い穴になった背景画像
- 間違った減算: テンプレートの残留 + ゴースト

全ケースで neg=0%（差分が負にならない）。サイノグラムが画像全体の線積分であるため、テンプレートのサイノグラム値は常に画像のサイノグラム値より小さい。

### 7.3 バイリニア補間版ラドン変換

`radonTransformInterp` として実装。`perpendicularSum`の`int()`丸めをfloat座標+バイリニア補間に置き換え。行シフトによる回転表現の精度を検証するために使用。本体の`radonTransformFloat`は既存のベンチマークとの互換性のため変更せず。
